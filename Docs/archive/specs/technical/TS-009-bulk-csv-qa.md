## Test Scenarios

### Tier 1 — Smoke (Issue #95): 5-URL end-to-end

**Input:** CSV with 5 URLs — mix of Manipal subpages and known-good external URLs.
**Environment:** Staging (Docker + cloudflared, per TS-008/ES-008) OR production with test account.

| # | Test case | Expected result |
|---|-----------|-----------------|
| S1 | Upload valid 5-URL CSV | 200 response, `jobId` returned, job row created in DB |
| S2 | Pipeline fan-out | 5 individual crawl tasks enqueued, not serial |
| S3 | Each URL reaches `completed` state | No URL stuck in `crawling` or `processing` after timeout |
| S4 | Audit output per URL | Report file present in Supabase Storage, non-empty |
| S5 | No Vercel function timeouts | No 504 or function kill in logs |
| S6 | Credit reserve → reconcile | Credits reserved on upload; reconciled per completed URL |
| S7 | Job overall state → `completed` | All 5 done → job marked complete |

**Pass criteria:** All 7 assertions pass. Any stuck URL = FAIL.

### Tier 3 — ZIP Integrity (Issue #97)

**Input:** Completed 5-URL job and completed 100-URL job from tiers above.

| # | Test case | Expected result |

**Pass criteria:** Z1–Z7 all pass. Z3 (partial failure) is critical — must handle gracefully.

## Edge Cases & Anti-Patterns to Test

| Edge case | Why it matters |
|-----------|---------------|
| CSV with duplicate URLs | Should deduplicate or handle gracefully — not double-bill credits |
| CSV with invalid/malformed URLs | Should skip with error row, not crash pipeline |
| CSV with 0 valid URLs | Validation error before job creation |
| URL that returns 404 during crawl | Crawl fails cleanly; record marked `failed`, not `stuck` |
| User cancels job mid-run | Job state transitions to `cancelled`; credits refunded |
| Concurrent upload by same user | Second upload either queues or rejects — no silent overwrite |
