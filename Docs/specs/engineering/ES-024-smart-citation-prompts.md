## a) Overview

Rewrites the AI Visibility citation prompt system from 4 static templates to 48 pillar-targeted prompts generated by a single `claude-haiku-4-5-20251001` call. Removes the citation check free path entirely. Increases batch concurrency from BATCH_SIZE=3/delay=500ms to BATCH_SIZE=20/delay=100ms.

### Current state

- `citation-prompt-generator.ts`: 23 lines, synchronous `generateCitationPrompts(site: { domain: string }, count: number = 4): string[]` ‚Äî returns 4 static template strings. Ignores geoScorecard.
- `citation-checker.ts`: `BATCH_SIZE=3` and `BATCH_DELAY_MS=500` hardcoded at lines 16-17. `runCitationCheck` takes `prompts: string[]` and returns no `pillarVisibility`.
- `citation-check/route.ts`: already gates on `creditBalance < CITATION_CHECK_COST`; no geoScorecard null check; calls synchronous `generateCitationPrompts(site)`.
- `lib/config.ts`: no citation batch constants.
- `lib/types/citation.ts`: `CitationCheckResult.scores` missing `pillarVisibility`; `prompt-generated` SSE event missing `pillar` field.

### What changes

5 files, no DB migrations:

| File | Change |
|------|--------|
| `geo/lib/services/citation-prompt-generator.ts` | Full rewrite ‚Äî async, 1 Haiku call, 48 `{ pillar, prompt }` objects, 4-prompt fallback |
| `geo/lib/services/citation-checker.ts` | Read BATCH_SIZE/DELAY from config; accept `{ pillar, prompt }[]`; add `pillarVisibility` to output |
| `geo/app/api/sites/[id]/citation-check/route.ts` | Hard 402 gate; add 422 for missing geoScorecard; await `generatePrompts()`; emit `pillarVisibility` |
| `geo/lib/config.ts` | Add `CITATION_CHECK_BATCH_SIZE=20`, `CITATION_CHECK_BATCH_DELAY_MS=100` |
| `geo/lib/types/citation.ts` | Add `pillarVisibility` to `CitationCheckResult.scores`; add `pillar` to `prompt-generated` SSE event |

#### 3d. Haiku system prompt

```
You are a search query generator. Generate natural-language questions that real users ask AI assistants
(ChatGPT, Perplexity, Google AI) to find or evaluate businesses.

For a given website, generate exactly 48 questions ‚Äî 3 per GEO pillar, 16 pillars total ‚Äî as a JSON array.

Rules:
1. Each question must sound like a real user query. Use natural language, not audit terminology.
2. Include the domain name in each question.
3. Tailor each prompt to its pillar dimension:
   - metadata_freshness    ‚Üí ask how recently content was updated on the site
   - semantic_html         ‚Üí ask if the site is well-structured for AI or search engines
   - structured_data       ‚Üí ask about schema markup and AI-readability of the site
   - entity_definitions    ‚Üí ask how the site defines key concepts, products, or entities
   - faq_coverage          ‚Üí ask about frequently asked questions the site answers
   - evidence_statistics   ‚Üí ask what data, research, or stats the site cites
   - content_structure     ‚Üí ask how content is organized or how easy it is to find answers
   - author_authority      ‚Üí ask about the expertise or credentials of the people behind the site
   - internal_linking      ‚Üí ask about how well related content is connected on the site
   - content_freshness     ‚Üí ask how current or up-to-date information on the site is
   - multi_format          ‚Üí ask if the site provides videos, infographics, podcasts, or other formats
   - licensing_signals     ‚Üí ask if the site's content is available or licensed for AI use
   - contact_trust         ‚Üí ask how to contact the site and whether it can be trusted
   - competitive_positioning ‚Üí ask how the site compares to alternatives or competitors
   - offering_clarity      ‚Üí ask what exactly the site offers or sells
   - cta_structure         ‚Üí ask how the site guides visitors to get started or take action
4. Consider siteType (e.g., SaaS, E-commerce, Agency, Blog) when phrasing questions.
5. Prioritize sharp, specific questions for pillars with score < 40.
6. Output ONLY valid JSON. No markdown, no explanation, no code fences.

Output format:
[
  { "pillar": "pillar_id", "prompt": "Natural language question?" },
  ...
]
```

{pillar.pillar} ({pillar.pillarName}): score={pillar.score}, findings="{pillar.findings.slice(0, 200)}"

If `geoScorecard` is null, substitute with: `"GEO Scorecard: not available ‚Äî generate general AI visibility prompts for all 16 pillars."`

#### 3f. Response validation

```typescript
function isValidPromptArray(data: unknown): data is { pillar: string; prompt: string }[] {
  return (
    Array.isArray(data) &&
    data.length === 48 &&
    data.every(
      (item) =>
        typeof item === "object" &&
        item !== null &&
        typeof (item as Record<string, unknown>).pillar === "string" &&
        typeof (item as Record<string, unknown>).prompt === "string"
    )
  );
}
```

If `data.length !== 48` or any item fails the check ‚Üí return fallback (don't throw).

## Risks

| Risk | Mitigation |
|------|-----------|
| Haiku returns fewer than 48 items | Validation rejects; fallback activates. |
| Haiku call takes >5s, slowing first SSE event | Haiku is called before streaming starts; p95 < 5s is expected. If consistently slow, add a 6s timeout wrapper around the Haiku call and fall back. |
| BATCH_SIZE=20 triggers provider rate limits | Conservative: 20 √ó ~2s avg + 0.1s delay ‚âà 20s for 192 calls. If rate limit errors appear, reduce to 15 in config. |
| `geoScorecard.pillars` has fewer than 16 entries (incomplete audit) | Haiku will generate fewer pillar buckets; pillarVisibility will have fewer keys. Acceptable. |
